# 新手能看懂的项目教程

这份教程只做一件事：带你把这个项目跑起来，并且让你知道每一步自己在干什么。

如果你现在还是大学生、第一次做这种 AI + 自动化 + 量化项目，按这份教程走就够了。

## 1. 先别管复杂概念，你先知道这个项目到底在做什么

你可以把整个系统理解成 4 个人在接力干活：

- 采集员：去不同网站把舆情数据拿回来
- 分析师：判断这些舆情是看多还是看空
- 交易员：根据信号做模拟买卖
- 报告员：把当天发生的事情整理成日报

所以这个项目不是一个单独脚本，而是一条流水线。

## 2. 你现在仓库里的代码，已经能做什么

当前版本已经可以：

- 跑通完整流程
- 自动生成日报
- 自动保存历史记录
- 自动做参数优化

但要注意一件事：

现在的数据源还是“演示用假数据”，不是你真的从股吧、微博、新浪抓回来的真实数据。

所以你当前阶段的目标不是一上来就做最复杂的功能，而是先：

1. 跑通项目
2. 看懂每个模块在做什么
3. 再把假数据替换成真实采集

## 3. 第一次运行项目

### 第一步：打开终端

你现在在 VS Code 里，直接打开 PowerShell 终端就行。

### 第二步：安装依赖

在项目根目录运行：

```powershell
./.venv/Scripts/python.exe -m pip install -r requirements.txt
```

如果已经装过了，这一步可以跳过。

### 第三步：运行一次日常流程

```powershell
./.venv/Scripts/python.exe run_pipeline.py --mode daily --date 2026-03-15
```

运行完之后，你会得到两类结果：

- 日报文件：`data/reports/2026-03-15.md`
- 历史记录：`data/memory/` 下面的几个文件

## 4. 运行后你应该去看哪里

你跑完以后，不要只看终端那几行字。你要学会看产物。

### 看日报

打开：

- `data/reports/2026-03-15.md`

你会看到：

- 最优平台组合
- 当天各股票的情绪快照
- 有没有交易信号
- 有没有模拟交易
- 当前账户现金和持仓

这份文件就是“报告员”的工作结果。

### 看历史记忆

打开：

- `data/memory/sentiment_history.jsonl`
- `data/memory/signal_history.jsonl`
- `data/memory/trade_history.jsonl`
- `data/memory/state.json`

它们分别表示：

- 舆情历史
- 信号历史
- 交易历史
- 当前账户状态

这部分非常重要，因为这就是你简历里可以写的“持久化记忆与状态管理”。

## 5. 这个项目的代码，应该怎么读

不要从头到尾乱看。按下面顺序读，最容易懂。

### 第 1 个文件：主入口

看：`run_pipeline.py`

它的作用很简单：

- 把 `src/` 加入 Python 路径
- 调用真正的主程序

你可以把它理解成“项目启动按钮”。

### 第 2 个文件：主程序

看：`src/opinion_trading/main.py`

它决定当前运行什么模式：

- `daily`：日常跑一次
- `backtest`：单次回测
- `optimize`：参数优化
- `visualize`：画图

你可以把这里理解成“控制台”。

### 第 3 个文件：工作流总控

看：`src/opinion_trading/agents/workflow.py`

这是全项目最核心的文件。

这里按顺序做了 4 件事：

1. 调采集员拿数据
2. 调分析师生成信号
3. 调交易员执行模拟交易
4. 调报告员生成日报

如果你只想快速理解项目，这个文件必须先看懂。

### 第 4 个文件：角色分工

看：`src/opinion_trading/agents/roles.py`

这里定义了 3 个角色：

- `CollectorAgent`
- `AnalystAgent`
- `TraderAgent`

它的意义不是复杂算法，而是“职责分开”。

以后你答辩时就可以说：

我把系统拆成多个角色，每个角色只做一层工作，这样方便替换、扩展和调试。

### 第 5 个文件：3 个核心 Skill

看这三个文件：

- `src/opinion_trading/skills/sentiment_collection.py`
- `src/opinion_trading/skills/sentiment_analysis.py`
- `src/opinion_trading/skills/trade_simulation.py`

它们分别对应：

- 采集
- 分析
- 模拟交易

这是你简历里最该重点讲的代码。

### 第 6 个文件：报告生成

看：`src/opinion_trading/core/report_builder.py`

这个文件负责把结果整理成 Markdown。

它对应“报告员”。

## 6. 你现在最该理解的 4 个概念

### 概念 1：什么叫 stub

`stub` 就是“临时假的实现”。

比如现在的：

- `src/opinion_trading/integrations/platform_sentiment_stub.py`

它不是去真实网站抓数据，而是随机生成演示数据。

为什么要这样做？

因为项目开发时，先用假数据把流程打通，比一开始就接真实网页更稳。

### 概念 2：什么叫持久化记忆

不是每次运行都从零开始，而是把历史记录存下来。

比如：

- 上次剩多少钱
- 上次持仓是什么
- 之前发过什么信号

这就是“有状态系统”，不是一次性脚本。

### 概念 3：什么叫多平台情绪共振

不是只看一个平台说涨说跌，而是同时看多个平台。

比如：

- 股吧很悲观
- 微博也很悲观
- 新浪财经也在变差

如果多个平台一起悲观，说明市场情绪更一致，这时候信号更强。

### 概念 4：什么叫反转信号

不是“悲观就买”。

而是：

- 前一天非常悲观
- 第二天情绪开始回升

这种“从极端悲观开始恢复”的过程，才叫反转。

## 7. 你接下来最应该做什么

只做一件事：把假数据换成真实采集。

不要同时做 10 件事。真正重要的顺序是：

### 第一步：先打通一个真实平台

推荐先做股吧。

目标不是一下做完全部平台，而是先证明你能：

- 打开网页
- 滚动页面
- 抓标题
- 抓时间
- 抓正文
- 保存成 CSV

### 第二步：再让分析师读真实文本

现在分析师吃的是已经存在的情绪分。

后面你要改成：

- 先读取文本
- 再调用 LLM 打分
- 再做平台聚合
- 再生成买卖信号

### 第三步：最后再做展示

等真实采集跑通以后，再补：

- 演示视频
- 图表
- README 美化
- 简历项目描述

今天不要贪多，按下面顺序做：

1. 再运行一次 `daily`

5. 打开 `platform_sentiment_stub.py`，理解为什么它现在还是假数据
6. 准备下一步把它替换成真实网页采集
